## コード構造分析

### アーキテクチャ
- Flutter + Firebase (Auth, Firestore, Storage, Messaging)
- 3層ロール: guest(未認証), user(一般), admin(管理者)
- MVPフェーズ: 管理者UIDがハードコード
- リポジトリパターン導入済み（JobRepository）
- ChatServiceにリトライ機能あり

### 主要な課題
1. **メッセージ機能のエラー**: frame_013で「再試行」ボタンが表示 → チャット初期化失敗
2. **管理者UIDハードコード**: AppConstants.adminUid = '5AeMBYb9PifYVUWMf4lSdCjuM1s1'
3. **状態管理**: setState()のみ使用、Provider/Riverpod等なし
4. **テストコードなし**
5. **CI/CDなし**
6. **デザインシステムなし**（ThemeDataは基本的なもののみ）
7. **メールログイン画面未実装**（TODOのまま）
8. **利用規約・プライバシーポリシー未実装**

### ファイル構成
- 合計約8,316行のDartコード
- lib/pages/ に直接UIコードが集中（一部 lib/presentation/ にも）
- lib/core/ にサービス・ユーティリティ
- lib/data/ にモデル・リポジトリ
- Firebase Cloud Functions (functions/index.js) あり

### 依存パッケージ
- firebase_core, firebase_auth, cloud_firestore, firebase_storage
- firebase_messaging
- image_picker, image
- url_launcher
- cupertino_icons
